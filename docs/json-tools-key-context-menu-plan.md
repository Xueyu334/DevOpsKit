# JSON Tools 键右键菜单功能记录

## 文档信息

- 创建日期：2026-04-15
- 目标页面：`src/views/json-tools/index.vue`
- 相关 Worker：`src/views/json-tools/json.worker.js`
- 状态：待排期

## 背景

`JSON 在线解析` 工具当前支持：

- 原始 JSON / JSON5 输入
- `String Parse (Strict)` 结果展示
- `JS Eval (Relaxed)` 结果展示
- 鼠标悬停查看路径
- 左键点击节点锁定路径
- 复制当前锁定路径

为了提升调试和排查效率，计划在结果区的对象键名上增加右键菜单能力，减少手动选中文本和人工拼路径的操作成本。

## 目标需求

当鼠标指向 `String Parse (Strict)` 或 `JS Eval (Relaxed)` 结果区中的某个对象键时，点击右键弹出菜单，提供以下操作：

1. 复制键名称
2. 复制值
3. 复制路径

## 范围说明

本次规划只针对 `json-tools` 页面内的结果区交互增强，不涉及：

- 路由调整
- 首页工具分类配置调整
- 全局右键菜单能力抽象
- 其它工具页复用改造

## 当前实现现状

### 页面层

`src/views/json-tools/index.vue` 当前已经具备以下基础能力：

- 通过 `v-html` 渲染 Worker 返回的 HTML 结果
- 在结果区容器上通过事件委托处理 `click` 和 `mouseover`
- 通过 `data-path` 标记当前节点路径
- 通过 `formatPath()` 将内部路径格式化为可展示路径
- 通过 `useClipboard()` 实现路径复制

这意味着右键菜单不需要重构当前渲染方式，可以沿用现有的事件委托模式继续扩展。

### Worker 层

`src/views/json-tools/json.worker.js` 当前已经具备：

- 解析输入内容并生成渲染 HTML
- 在对象键和部分值节点上输出 `data-path`
- 支持通过路径读取节点值
- 支持局部加载大对象 / 大数组内容

这意味着“复制值”不必从 DOM 文本反推，更稳妥的方式是继续由 Worker 按路径回查原始解析结果。

## 可行性结论

该功能可行，且改动范围较集中，主要集中在：

- `src/views/json-tools/index.vue`
- `src/views/json-tools/json.worker.js`

基于当前工具的产品语义，可采用以下前提：

- 合法标准 JSON 输入时，Strict 与 Relaxed 两列结构和值应保持一致
- 非法标准 JSON 但可被 Relaxed 解析时，Strict 列不会出现可交互的键节点

因此右键菜单的“复制值”能力可以继续基于当前 Worker 最近一次成功解析结果进行查询，不必为了该功能额外拆分两套缓存模型。

## 推荐实现方案

### 1. 结果区增加右键事件委托

在两个结果列容器上增加 `@contextmenu.prevent` 处理函数：

- 仅当命中对象键节点时弹出菜单
- 命中其它区域时不弹出菜单
- 不影响现有左键点击锁定路径逻辑

### 2. 为键节点补充稳定的元数据

建议在 Worker 输出对象键节点 HTML 时保留或补充以下信息：

- `data-path`
- `data-key-name`
- 可选：`data-node-type="key"`

这样主线程可以稳定拿到：

- 当前键名称
- 当前节点路径
- 当前节点是否为“可弹菜单的键节点”

避免依赖 DOM 文本提取，减少引号、转义符和样式标签导致的误判。

### 3. 页面内实现轻量菜单浮层

建议在 `index.vue` 内实现一个轻量自定义菜单，而不是引入复杂封装：

- 使用响应式状态控制显示与隐藏
- 根据鼠标坐标定位到右键触发位置
- 点击菜单项后执行复制动作并自动关闭
- 点击空白、滚动、按 `Esc` 时关闭菜单

菜单项固定为：

- 复制键名称
- 复制值
- 复制路径

### 4. 复制逻辑建议

#### 复制键名称

优先使用 `data-key-name`。

#### 复制路径

复用页面现有 `formatPath()` 逻辑，保持与底部路径栏一致，输出格式建议为：

```text
$.user.profile.name
```

或在遇到特殊键名时输出：

```text
$["user.info"][0]["display-name"]
```

#### 复制值

推荐新增 Worker 消息类型，例如：

- `inspect`
- `getValue`

主线程传入当前节点 `path`，Worker 负责：

- 通过现有路径读取逻辑定位值
- 将值转换为适合复制的文本
- 返回主线程，再由 `useClipboard()` 写入剪贴板

## 复制值的建议规则

为了保证复制结果符合用户直觉，建议采用以下规则：

- 字符串：复制原始字符串内容，不带展示引号
- 数字：复制数字字面量
- 布尔值：复制 `true` / `false`
- `null`：复制 `null`
- `undefined` / `NaN` / `Infinity`：按可见字面量复制
- 对象 / 数组：复制格式化后的 JSON 字符串

如果后续希望与界面展示完全一致，也可以改为：

- 字符串复制时保留引号

该点属于产品细节，开发前建议最终确认一次。

## 交互建议

### 触发方式

- 仅对象键名支持右键菜单
- 普通值节点不弹菜单
- 折叠按钮、加载更多按钮不参与右键菜单逻辑

### 菜单关闭时机

- 点击菜单外区域
- 执行任一菜单动作后
- 页面滚动
- 按下 `Esc`
- 数据重新渲染后

### 反馈方式

建议继续复用 `ElMessage`：

- 复制键名称成功
- 复制值成功
- 复制路径成功

## 对现有逻辑的影响评估

### 低风险

- 悬停路径展示
- 左键锁定路径
- 路径复制按钮
- Strict / Relaxed 渲染

### 需要注意

- 右键事件不要误触发左键锁定逻辑
- 右键菜单定位要避免超出结果区或视口边界
- 局部加载内容插入后，新节点也需要自然支持右键事件

由于当前使用的是容器级事件委托，只要菜单逻辑同样走委托方式，局部加载的新节点无需额外绑定事件。

## 验收建议

### 功能验证

- 合法 JSON 下，Strict 和 Relaxed 区对象键右键均能弹菜单
- 点击“复制键名称”后，剪贴板内容正确
- 点击“复制值”后，剪贴板内容正确
- 点击“复制路径”后，剪贴板内容正确
- 点击空白区域，菜单关闭
- 局部加载后的新节点右键仍可用

### 边界验证

- 特殊字符键名可正确复制
- 空字符串键名可正确复制路径
- 数组中的对象键可正确复制路径和值
- 非法 JSON 导致 Strict 区报错时，不出现异常菜单
- 大对象 / 大数组场景下，右键菜单不影响现有性能表现

## 备注

当前该功能属于体验增强项，适合在后续迭代中按最小改动方式接入。  
如果未来希望支持更复杂的节点操作，例如：

- 复制完整子树
- 展开到此节点
- 跳转到输入区定位
- 收藏常用路径

建议届时再考虑将右键菜单抽成独立的局部组件或 composable。
